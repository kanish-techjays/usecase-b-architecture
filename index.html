Use Case B · Job Summarisation Copilot
Technician flow today · technician flow with the AI copilot · high-level architecture · low-level process flow

1. Technician User Flow · Today (without AI)

The current experience in the mobile app. Every step works, but the tech goes in cold, types everything by hand, and problems surface only after they have left the site. Red = where it hurts today.

2. Technician User Flow · With the AI Copilot - same journey, every pain point answered

Top lane: what the technician does. Bottom lane: what the AI does for them at that moment. Blue = tech action · amber = AI-assisted step.

The copilot moments in words: at assignment, the briefing builder pulls this site's history from SQL and similar incidents from the vector index, and Gemini writes a short card ("the previous techs did X; the issue recurred; try Y first"), cached so it opens instantly. In the wizard, known values are prefilled so the tech only types what is new. On Review, the analyse pipeline returns the 12-section ticket and procedure checklist; gaps are fixed inline until clean, and the accepted summary is persisted, becoming the history that briefs the 11th tech.

3. High-Level Architecture

Read left to right: the app talks only to the existing .NET APIs; the Microservices API proxies AI requests to our private FastAPI server, which calls Gemini, persists everything to SQL Server, and pulls similar past incidents from the vector index, which is seeded with the last 12 months of history and updated the moment each new dispatch completes.

4. Low-Level Architecture · End-to-End Process Flow

One request, followed top to bottom through every component: who acts (color-coded), what each module does, and what flows across each arrow. Statuses are honest: Built vs Planned.
